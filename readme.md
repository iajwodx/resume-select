# 简历筛选系统

## 功能需求

### 1. 简历获取和分析

- 爬虫或者用户提供简历
- 接入AI对简历进行分析

### 2. 简历筛选

#### 筛选方法

**第一步筛选111：**

- 期望工作地点（多选，符合其一即可）
- 工作年限（大于某年限）
- 学历（可多选，符合其一即可）
- 期望薪资（选择范围，应该与简历有交集）
- 求职状态筛选（单选）

> 每项中的多选（如地点：上海，北京。则符合其一即可）都至少符合一个。

**第二步筛选：**

- 关键词搜索，提供文本匹配度

#### 筛选结果处理

- 不同的筛选条件用不同的高亮来展示

#### 排序方法

1. 根据文本匹配度进行排序
2. 根据简历最近的更新时间进行排序

## UI 设计

- **登录**
- **简历获取：**
  - 简历爬取或接入API
  - 简历上传
  - 展示获取的简历（姓名，联系方式，期望工作地点，工作年限，学历，期望薪资，技能，项目经验，简历更新时间）
  - 可对简历进行编辑
- **简历筛选：**
  - 期望工作地点（多选，符合其一），工作年限（大于某年限），学历（可多选，符合其一），期望薪资（选择范围，应该与简历有交集），求职状态筛选（单选）进行第一步筛选。关键词进行第二步筛选。
  - 筛选结果：展示的是缩减的内容。可通过详情获得完整简历。
  - 可手动对结果的部分不合格简历进行隐藏，或者可以收藏简历（标识可能岗位）。
  - 不同的筛选条件用不同的高亮来展示。
  - 排序方法：1.根据文本匹配度进行排序。2.根据简历最近的更新时间进行排序。
- **收藏简历展示：**展示简历以及标识的岗位。

## 后端

框架选择：Spring Boot，Java。

- **登录模块：**数据库存储用户和密码。（暂时为use123,密码123456）
- **简历获取模块：**获取简历PDF → PDFBox 提取文本 → DeepSeek AI 分析提取结构化信息 → 存入数据库（名字和联系方式一致的合并为更全面的） → 读取数据库展示已有简历。编辑简历。
- **简历筛选模块：**接收筛选规则，关键词用AI匹配获取匹配度，其余的用文本匹配，与对应部分进行文本匹配，匹配到的需要高亮。排序：通过获取的匹配度排序和更新时间。
